## MyBatis resultMap 매핑으로 조회 결과가 1건만 반환되던 문제

### 문제
상품 목록 조회 시 SQL상으로는 여러 건이 조회되었지만, 실제 응답에서는 1건만 반환되는 문제가 있었습니다.  
조회 결과가 의도치 않게 하나의 객체로 합쳐지면서 목록 데이터가 정상적으로 표현되지 않았습니다.

### 해결
원인을 확인한 결과, `resultMap`에서 `<id>`를 지정하지 않아 MyBatis가 각 행을 구분하지 못하고 하나의 객체로 병합하고 있었습니다.  
처음에는 식별용 필드를 DTO에 추가하는 방법도 검토했지만, 응답 객체에 불필요한 필드를 추가하지 않기 위해 조회 목적에 맞게 DTO 구조를 단순화하고 필요한 값만 명확히 매핑하도록 수정했습니다.

### 결과
조회 결과가 각 행 단위로 정상 매핑되면서 상품 목록이 올바르게 반환되도록 개선했습니다.  
이 경험을 통해 `resultMap`의 `<id>`는 결과 객체를 식별하고 중복 병합을 방지하는 중요한 기준이라는 점을 알게 되었습니다.
