## JSP `form:form` 바인딩 기준을 잘못 이해해 path 지정이 잘못된 문제

### 문제
JSP 폼 화면 구현 과정에서 `form:form`을 사용할 때 바인딩이 정상적으로 동작하지 않는 문제가 있었습니다.  
`modelAttribute`로 이미 기준 객체가 지정되어 있었는데도, 일반 EL 표현식처럼 객체명을 한 번 더 포함해 `path`를 작성하면서  
필드 바인딩이 올바르게 이루어지지 않았습니다.

### 해결
원인을 확인한 결과, `form:form` 내부의 `path`는 `modelAttribute`를 기준으로 필드명만 작성해야 했습니다.  
따라서 객체명을 포함한 경로 대신 필드명만 사용하도록 수정해 바인딩 기준을 맞췄습니다.

### 결과
폼 화면이 정상적으로 렌더링되고, 입력값 바인딩도 의도한 대로 동작하도록 개선했습니다.  
이 경험을 통해 JSP의 `form:form` 태그는 일반 EL과 참조 방식이 다르며,  
`modelAttribute`가 기준 객체 역할을 한다는 점을 알게 되었습니다.
